# Cycle Diagram Guide

這份文件對照 `project/config/image_templates/cycle_diagram.json`，說明目前穩定版的：

- path 幾何設定
- marker / guides / overlay_text 的設定位置
- 目前已實作的 named anchors
- 未來擴充時的建議方向

## 1. 目前實作範圍

目前 cycle diagram 的主要輸出流程如下：

- `generate_image`
  - 外框
  - 左側短刻度
  - cycle path
  - marker
  - guides
- `overlay_text`
  - 溫度、日期、循環次數、legend 等文字

這份文件只描述 repo 目前真的存在的設定與命名，不把未來規劃混進「目前可用」。

## 2. path_builder 內的三種 block

目前 path 幾何使用三種 block：

- `outer_first_block`
  - 第一個循環
- `inner_block`
  - 中間循環
- `outer_last_block`
  - 最後一個循環與收尾

### `outer_first_block`

目前結構：

```text
rise -> outer_high
hold -> outer_high
fall -> inner_high
hold -> inner_high
fall -> outer_low
hold -> outer_low
rise -> inner_low
hold -> inner_low
```

### `inner_block`

目前結構：

```text
rise -> inner_high
hold -> inner_high
fall -> inner_low
hold -> inner_low
```

### `outer_last_block`

目前結構：

```text
rise -> outer_high
hold -> outer_high
fall -> inner_high
hold -> inner_high
fall -> outer_low
hold -> outer_low
rise -> inner_low
hold -> inner_low
rise -> a_bit_high_then_start
hold -> a_bit_high_then_start
fall -> start
hold -> start
```

## 3. 哪些 config 用來改 path 幾何

如果要改線，優先看 `path_builder`：

- `levels`
  - 控制高低平台的垂直位置
- `left_lead_segments`
  - 控制左側起始段
- `outer_first_block`
  - 控制第一段 outer 幾何
- `inner_block`
  - 控制中間 inner 幾何
- `outer_last_block`
  - 控制最後一段 outer 與收尾
- `right_tail_segments`
  - 控制最尾端的最小延伸責任

### 快速對照

- 想改高低差：看 `levels`
- 想改左側起始節奏：看 `left_lead_segments`
- 想改第一段 outer：看 `outer_first_block`
- 想改中間 inner：看 `inner_block`
- 想改最後收尾：看 `outer_last_block`
- 想改最尾端最後一小段：看 `right_tail_segments`

## 4. 哪些 config 用來改 marker / guides / overlay_text

`cycle_diagram.json` 目前可分成三層：

- `markers`
  - 起終點與線上 marker
- `guides`
  - 虛線、箭頭、區間標註
- `fields`
  - overlay_text 文字內容

快速對照：

- 想改 marker 大小或位置：改 `markers`
- 想改灰色虛線、箭頭、AB 標註：改 `guides`
- 想改溫度、日期、legend、循環文字：改 `fields`

## 5. 目前可用的 anchor 類型

marker 目前可用的 `anchor` 有三種：

- `start`
  - path 起點
- `end`
  - path 終點
- `named_anchor`
  - 用 `anchor_ref` 指向已命名的幾何點

### 什麼時候用 `start` / `end`

當你要掛的是左右兩端固定資訊時，用：

- `start`
- `end`

### 什麼時候用 `named_anchor`

當你要掛的是線上的特定位置，而且希望 marker 跟著 path 幾何移動時，用：

- `anchor: "named_anchor"`
- `anchor_ref: "<anchor name>"`

不要用 `start + dx/dy` 或 `end + dx/dy` 硬推一個其實屬於線上轉折點的位置。

## 6. 目前實際可用的 named anchors

以下清單已和目前 `generate_image` 的 debug 輸出對齊。

### 固定存在

- `path_start`
- `lead_kink`
- `lead_end`
- `outer_1_high`
- `outer_1_low`
- `outer_2_high`
- `outer_2_low`
- `tail_start`
- `path_end`

### 依 `cycle_count` 增加

- `inner_1_high`
- `inner_1_low`
- `inner_2_high`
- `inner_2_low`
- `inner_3_high`
- `inner_3_low`
- ...

命名規則是：

- `inner_<index>_high`
- `inner_<index>_low`

## 9. 想加 marker 時怎麼判斷

### 情況 A：現有 `named_anchor` 已夠用

例如要掛在：

- `lead_kink`
- `outer_2_low`
- `inner_1_high`
- `tail_start`

就直接在 `markers` 裡使用 `named_anchor`。

### 情況 B：現有 anchor 不夠

如果要掛的是新的平台或新的轉折點：

1. 先補 path builder 的 `named_anchor`
2. 再回 `markers` 加 `anchor_ref`

不要直接退回手寫 pixel 座標。

## 10. 後續擴充建議順序

建議順序：

1. 先補 `named_anchors`
2. 再補更多轉折點
3. 最後才考慮更細的 anchor 模式

不建議直接走成通用圖形引擎或任意座標系統。
